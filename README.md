# STM32F407 电缆长度与终端负载测量装置

本项目基于 STM32F407ZGT6、TDC-GP22、62 Ω直流分压电路和 NE555 周期测量电路，实现：

- 电缆长度原始飞行时间测量；
- 终端电阻测量；
- 终端电容与开路分类；
- Nextion/HMI 显示和 USART3 原始数据调试。

当前软件中的长度、电阻和电容系数借自另一套成功硬件，仅用于验证计算链，所有相关日志均带有 `REF_SIM` 标识。它们不是本机最终标定参数，不能用于宣称测量精度。

## 1. 使用前必读

1. GP22、NE555、继电器控制端和 STM32 必须共地。
2. GP22 模块必须使用 3.3 V，禁止接入 5 V 或原图中已经废弃的 6 V。
3. PA1、PE0、PA4～PA7、PE1、PE9 上的信号不得超过 STM32F407 的电气规格。
4. PG2、PG3、PG4 只能连接继电器驱动器，不能直接驱动继电器线圈。
5. 标准双极型 NE555 通常不保证在 3.3 V 工作，必须确认实际器件是支持低压的 CMOS 555，或已经实测能在 3.3 V 稳定振荡。
6. 当前长度节点中 GP22 的 `STA`、`SP1` 和 STM32 的 TIM1 脉冲共节点。若 10 m、20 m 的 GP22 原始值没有稳定差异，应停止调整软件公式，优先检查共节点自触发和模拟前端整形。

## 2. 软件结构

```text
Core/
  Src/main.c                 外设初始化、主循环和中断统一转发
User/
  app/app_process.c          HMI命令、A/B同步测量任务、调试日志
  bsp/bsp_path.c             三路继电器互锁
  bsp/bsp_length.c           GP22硬件SPI、复位、单次原始测量
  bsp/bsp_load.c             ADC原始采样、TIM5周期捕获
  alog/alog_measure.c        平均值和参考模拟公式
MDK-ARM/
  TI_CQUPT_T3.uvprojx        Keil MDK工程
TI_CQUPT_T3.ioc              CubeMX硬件配置
```

BSP 只返回硬件原始量，ALOG 负责平均和公式，APP 负责继电器切换、测量顺序、HMI 与 USART3 日志。中断回调中不进行 SPI、UART、浮点计算或阻塞延时。

## 3. 快速使用手册

### 3.1 编译与烧录

1. 使用 Keil μVision 5 打开 `MDK-ARM/TI_CQUPT_T3.uvprojx`。
2. 选择目标 `TI_CQUPT_T3`。
3. 执行 `Rebuild all target files`。
4. 当前基准应为 `0 Error(s), 0 Warning(s)`。
5. 使用 ST-Link 下载到 STM32F407ZGT6。
6. 复位后确认三个继电器均释放，HMI 的 `status` 显示 `Ready`。

当前工程使用 ARM Compiler 5.06，主频为 168 MHz。

### 3.2 调试串口

USART3 用于电脑调试：

| 参数 | 设置 |
| --- | --- |
| 波特率 | 115200 bit/s |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | None |
| 流控 | None |
| MCU TX | PB10 |
| MCU RX | PB11 |

USB-TTL 必须使用 3.3 V逻辑，连接时 TX/RX 交叉，并与主板共地。

### 3.3 HMI 操作

USART1 连接 HMI，通信参数为 9600-8-N-1。页面必须存在以下文本控件：

| 控件 | 内容 |
| --- | --- |
| `status` | Ready、Measuring、SIM Hold、Measure L First、Error |
| `len` | 最近一次参考模拟长度 |
| `kind` | R、C、OPEN、ERROR |
| `para` | 电阻或电容参考模拟值 |

HMI 向 USART1 发送单字节命令：

| 命令 | 功能 |
| --- | --- |
| `A` | 执行长度任务 |
| `B` | 执行负载任务 |

推荐顺序：

1. 发送 `A`，取得有效长度。
2. 确认 USART3 的 GP22 原始值可信。
3. 连接待测负载后发送 `B`。
4. 读取 `kind`、`para`，同时保存 USART3 原始日志。

若未先成功执行 `A` 就发送 `B`，程序仍会输出 ADC 和 NE555 原始量，但不会进行电缆寄生补偿，HMI 显示 `Measure L First`。

### 4.3 电阻分压连接

当前代码只对应以下接法：

```text
3.3 V ── 62 Ω参考电阻 ── PC0/ADC节点 ── 被测负载 ── GND
```

计算关系：

```text
Vadc  = 3.3 × Rload / (62 + Rload)
Rload = 62 × Vadc / (3.3 - Vadc)
```

若实物上下臂接反，ADC 会呈现相反趋势，不能通过修改校准系数掩盖。

### 4.4 NE555 电容测量连接

| NE555引脚 | 连接 |
| --- | --- |
| Pin1 GND | GND |
| Pin2 TRIGGER | 与 Pin6、电容节点C相连 |
| Pin3 OUT | PA1/TIM5_CH2 |
| Pin4 RESET | 3.3 V |
| Pin5 CONTROL | 10 nF到GND |
| Pin6 THRESHOLD | 与 Pin2、电容节点C相连 |
| Pin7 DISCHARGE | R1与R2连接点 |
| Pin8 VCC | 3.3 V |

定时网络：

```text
3.3 V ── R1 10 kΩ ── Pin7 ── R2 1 MΩ ── 电容节点C
电容节点C ── 被测总电容 ── GND
```

理想无稳态频率：

```text
f = 1.442695 / ((R1 + 2 × R2) × C)
```

TIM5 输入时钟为84 MHz，PSC=41，因此捕获 Tick 为2 MHz，每个 Tick 为0.5 μs。

## 5. 软件测量流程和日志

### 5.1 长度任务

发送 `A` 后：

1. 关闭全部继电器，再只吸合 PG2。
2. 复位 GP22，写入7个固定配置寄存器。
3. 读取 Reg1 高字节，期望值为 `0x01`。
4. 连续尝试100次测量，每次超时20 ms。
5. 每次由 PE9 产生一个 TIM1 单脉冲。
6. 等待 PE0 的 GP22 INTN下降沿。
7. 读取 STAT 和 RES0。
8. 至少成功10次后，对所有成功 RES0 做有符号平均。
9. 输出 raw 和参考模拟长度，最后释放继电器。

主要日志：

```text
[L_RAW] ok=1 raw=0x........ status=0x....
[L_RAW] fail=N
[L_REF_SIM] fixed=... time_ns=... length_cm=...
[L_SUM] success=... samples=... failures=... raw_avg=... elapsed_ms=...
```

`[L_RAW]` 是判断 GP22 和前端是否工作的首要依据；在 raw 没有通过10 m/20 m对比前，不应讨论参考长度是否准确。

### 5.2 负载任务

发送 `B` 后：

1. 只吸合 PG3。
2. 丢弃前4次 ADC，再采集100次。
3. 输出 ADC 最小值、最大值、平均值和电压。
4. 有有效长度时，若参考电阻结果在0～100 Ω内，判定为 `R` 并结束。
5. 否则释放 PG3，吸合 PG4。
6. 捕获32个 NE555 周期，最大等待1 s。
7. 输出周期最小值、最大值、平均值和频率。
8. 有有效长度时，参考电容不小于30 pF判定为 `C`，否则显示 `OPEN`。
9. 任意退出路径最终都会释放三路继电器。

主要日志：

```text
[R_RAW] min=... max=... avg=... voltage=...
[R_REF_SIM] r_calc=... cable_r=... r_load=...
[C_RAW] period_min=... period_max=... period_avg=... tick_hz=2000000 freq=...
[C_REF_SIM] freq=... total=... cable=... load=...
[LOAD_SUM] success=... elapsed_ms=...
```

## 6. 硬件确认操作指南

硬件验证必须按下面顺序进行。前一步不通过时，不要跳到后面的标定。

### 6.1 断电与供电检查

断电状态：

- 用万用表确认3V3与GND无明显短路。
- 核对 U4/U5 Pin1方向。
- 核对继电器线圈有驱动三极管/MOSFET和续流保护。
- 核对 PC0 确实连接在62 Ω与负载之间。
- 核对 PA1 直接连接 NE555 Pin3，而不是定时电容节点。

上电状态：

- 测量 MCU 3V3、GP22 VCC和555 VCC，均应约为3.3 V。
- 确认 PE1 和 PA4 空闲时为高电平。
- 确认 PG2、PG3、PG4 复位后均为低。
- 检查模块温升和电源电流；异常时立即断电。

### 6.2 继电器互锁

建议使用三通道逻辑分析仪同时观察 PG2、PG3、PG4：

1. 发送 `A`：只能看到 PG2 为高。
2. 发送 `B` 且尚无有效长度：先看到 PG3 为高，随后全低，再看到 PG4 为高。
3. 每次切换中间应存在约10 ms三路全低区间。
4. 任务结束后三路必须全部为低。
5. 任意时刻若两路同时为高，停止连接测量前端并检查 GPIO/继电器驱动。

还应在断电状态用万用表通断档检查触点，确认“GPIO编号正确”并不只是驱动指示灯正确。

### 6.3 GP22 SPI和复位

用逻辑分析仪观察 PA4～PA7、PE1：

1. PE1 先低约1 ms，再回到高。
2. PA4 每个 SPI事务期间为低，事务之间为高。
3. 先出现 `0x50` Power-On Reset。
4. 随后出现7条四字节配置命令。
5. 读命令 `0xB5` 后，PA6 返回值应为 `0x01`。

如果 APP 输出 `[L_RAW] init_or_path_failed`：

- 先检查 GP22 是否为3.3 V供电和共地；
- 检查 PA6/PA7 是否接反；
- 检查 PA4 是否确实连接 U5 Pin6；
- 检查 SPI模式是否为 CPOL=0、CPHA=1；
- 检查 PE1 是否被外部电路持续拉低。

### 6.4 TIM1脉冲与GP22中断

示波器同时观察 PE9 和 PE0：

- PE9 每次测量应出现一次约1 μs的脉冲；
- PE9 高电平宽度约0.5 μs；
- PE0 空闲为高，完成时产生下降沿；
- PE0不得长期保持低电平。

PE9 只是逻辑触发，不能直接驱动50 Ω同轴电缆。若接入同轴后边沿严重变慢，应检查外部驱动、阻抗匹配、比较器和保护网络。

### 6.5 10 m/20 m长度原始值测试

使用同类型、已知长度的10 m和20 m电缆，每条至少重复5次 `A` 命令，记录：

| 电缆 | 第1次raw_avg | 第2次 | 第3次 | 第4次 | 第5次 | 变化趋势 |
| --- | --- | --- | --- | --- | --- | --- |
| 10 m |  |  |  |  |  |  |
| 20 m |  |  |  |  |  |  |

通过条件：

- 每轮至少10个成功样本；
- 同一长度的 raw 不应随机跳变或长期固定；
- 10 m与20 m raw 应有稳定、可重复且方向一致的差异；
- 多次插拔后趋势仍一致。

立即停止算法调整的条件：

- raw 始终接近0；
- raw 固定不变；
- 10 m与20 m没有可重复差异；
- 长度增加时 raw 变化方向随机；
- PE9正常但 PE0 无有效响应。

出现上述情况时，优先报告并检查“START/STOP共节点或前端整形问题”。不得修改比例系数伪造长度差异。

### 6.6 ADC和62 Ω分压

先不依赖参考模拟电阻结果，只查看 `[R_RAW]`。分别连接短路、10 Ω、20 Ω、30 Ω和开路负载。

理想值仅用于核对接法方向：

| 负载 | 理想Vadc | 理想12位ADC码 |
| --- | ---: | ---: |
| 短路 | 0 V | 0 |
| 10 Ω | 0.458 V | 约569 |
| 20 Ω | 0.805 V | 约999 |
| 30 Ω | 1.076 V | 约1336 |
| 开路 | 接近3.3 V | 接近4095 |

通过条件：

- ADC平均值随负载电阻增大而单调增大；
- `min/max`跨度较小且重复测量稳定；
- PC0实测电压与串口换算电压接近；
- 开路和短路方向与上表一致。

若趋势相反，应检查62 Ω与负载是否接反。若波动大，应检查继电器触点、地线、ADC节点走线和电源去耦，不要先修改增益。

### 6.7 NE555与TIM5周期

先用示波器直接观察 PA1：

- 必须存在持续、稳定的0～3.3 V兼容方波；
- 频率应随总电容增大而下降；
- 占空比可以不为50%，软件只测相邻上升沿；
- 不应出现停振、双边沿或幅值超过MCU规格。

R1=10 kΩ、R2=1 MΩ且忽略电缆寄生时，理想参考如下：

| 独立测试电容 | 理想频率 | 2 MHz下理想周期Tick |
| --- | ---: | ---: |
| 100 pF | 约7.18 kHz | 约279 |
| 200 pF | 约3.59 kHz | 约557 |
| 300 pF | 约2.39 kHz | 约836 |

实际接入电缆后，总电容包含电缆、PCB、继电器和探头寄生，周期会大于上表。

若 PA1 无稳定方波：

1. 确认实际芯片支持3.3 V；
2. 检查 Pin4 RESET 是否为高；
3. 检查 Pin2与Pin6是否相连；
4. 检查 R1、R2阻值和焊接；
5. 检查 Pin5的10 nF；
6. 检查示波器探头电容是否明显影响节点。

在 PA1 波形正常之前，不要继续修改 TIM5 软件。

### 6.8 HMI复测

1. 上电后四个文本控件均有初始值。
2. `A` 能进入 `Measuring`，结束后显示 `SIM Hold`或`Error`。
3. `B` 能更新 `kind`和`para`。
4. 无长度时显示 `Measure L First`。
5. 制造一次GP22或NE555超时后，恢复接线并再次发送命令，系统应能重新测量。

## 7. 本机标定计划

只有原始量验证通过后才能标定。

### 7.1 长度

使用10 m、15 m、20 m同型号电缆，每个长度至少采集10组稳定 `raw_avg`：

```text
raw_avg -> 实际长度cm
```

拟合本机斜率、截距和空载基准。先检查残差和单调性，再替换参考模拟参数。

### 7.2 电阻

使用经过万用表实测的10 Ω、20 Ω、30 Ω标准负载：

- 记录 ADC min/max/avg；
- 同时记录 PC0 实测电压；
- 使用实际62 Ω阻值而不是只用标称值；
- 分别估算电压误差、分压换算误差和每厘米电缆电阻。

### 7.3 电容

使用经过电桥/LCR表确认的100 pF、200 pF、300 pF标准电容：

- 记录 period_min/period_max/period_avg；
- 记录示波器频率；
- 分离空板、继电器、电缆和负载寄生；
- 重新拟合频率及总电容修正系数。

标定完成前必须保留 `ALOG_REFERENCE_SIMULATION_ONLY=1` 和所有 `REF_SIM`日志标识。

## 8. 常见故障

| 现象 | 优先检查 |
| --- | --- |
| `[L_RAW] init_or_path_failed` | GP22供电、SSN、MISO/MOSI、RSTN、Reg1读回 |
| 100次长度全部超时 | PE9脉冲、PE0中断、GP22 START/STOP前端 |
| 10 m/20 m raw无差异 | STA/SP1共节点自触发、比较器/整形、同轴连接 |
| ADC始终接近0 | PC0短路、负载短路、62 Ω未接3.3 V |
| ADC始终接近4095 | 负载开路、PC0未接负载、地线断开 |
| ADC趋势与理论相反 | 分压上下臂接反 |
| `[C_RAW]`超时 | 555不起振、PA1接错、RESET低、TIM5无边沿 |
| 周期偶发极小值 | PA1毛刺、地弹、输入边沿振铃、CC2OF |
| HMI无显示但USART3正常 | USART1 TX/RX、9600波特率、控件名或页面 |
| 任务失败后继电器仍吸合 | 驱动级极性、继电器硬件粘连；软件输出应已拉低 |

## 9. 当前限制

- 所有物理量仍是参考模拟结果。
- 软件不能判断 GP22 共节点捕获的是发射边沿还是反射边沿。
- 软件不能修复3.3 V下不起振的双极型 NE555。
- 当前平均算法不剔除异常值，原始链路稳定性应先由硬件保证。
- 当前负载分类阈值仅用于趋势验证，不代表最终比赛判据。
- 最终比赛前必须完成本机长度、电阻和电容三组重新标定。
