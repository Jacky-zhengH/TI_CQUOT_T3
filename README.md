# STM32F407 同轴电缆长度与终端负载检测装置

本工程使用 STM32F407ZGT6、STM32 HAL 和 CubeMX，实现三条互斥测量通路：

- TDR + TDC-GP22 测量电缆长度；
- 直流分压 + ADC 测量终端电阻；
- NE555 振荡 + TIM5 输入捕获测量终端电容。

HMI 发送 `A` 启动长度检测，发送 `B` 启动负载检测。每次检测的 APP 总超时均为 5 s。

## 1. 程序采集逻辑

初始化顺序为：

```text
CubeMX外设初始化
  -> bsp_path_init()
  -> bsp_load_init()
  -> bsp_length_init()
  -> HMI_Process_Init()
  -> App_Main_Process_Poll()
```

APP 负责完整测量流程；BSP 只控制硬件并返回原始数据；ALOG 负责统计、标定换算和负载分类。所有流程退出时都会停止采集并关闭三个继电器。

### 1.1 长度检测

1. 选择长度继电器 PG2，先全断 10 ms，再接通并等待 10 ms。
2. 每次向 GP22 发送 `INIT`，由 PE9 输出一次单脉冲。
3. 等待 GP22 的 INTN 下降沿，单次最长等待 20 ms。
4. 检查 GP22 状态寄存器，再读取 RES0 有符号 16.16 原始值。
5. 最多尝试 48 次，目标取得 16 个有效值，至少需要 8 个。
6. 将有效值排序，去掉最低 2 个和最高 2 个，再对中间数据求平均。
7. 使用 Flash 标定参数换算：

```text
长度(m) = (length_gain_cm_per_raw × raw平均值
           + length_offset_cm) / 100
```

8. 长度有效范围限制为 0.1～30 m。
9. 随后切换到电容通路，捕获 16 个 NE555 周期，保存为该电缆的“开路周期基准”。

当前代码没有直接使用传播速度和 `L = v × t / 2` 公式，而是使用实测线性标定系数。

PE9 的 TIM1 参数为 168 MHz 计数、1 μs 单脉冲周期、前 500 ns 为高电平。PE9 只是 3.3 V 逻辑触发信号，不能直接驱动 50 Ω 同轴电缆。

### 1.2 电阻检测

1. 必须先有一次成功的长度检测结果和开路周期基准。
2. 选择电阻继电器 PG3。
3. ADC1 在 PC0 上采用软件触发、12 位、右对齐、84 周期采样。
4. 先丢弃 4 次转换，让分压节点和 ADC 采样电容稳定。
5. 再采集 32 个 ADC 原始码，计算最小值、最大值和算术平均值。
6. 当前软件按以下分压接法计算，参考电阻标称值为 62 Ω：

```text
3.3 V ── 62 Ω参考电阻 ── PC0 ── 被测负载 ── GND

ratio = ADC平均值 / 4095
分压所得电阻 = 62 × ratio / (1 - ratio)
最终电阻 = gain × 分压所得电阻 + offset
           - cable_resistance_per_m × 电缆长度
```

7. 计算结果在 5～50 Ω 内时，直接判定为电阻负载。
8. ADC 码靠近 0 或 4095 时拒绝换算，避免开路、短路附近除法失稳。

代码虽然已经保存 62 Ω 默认值，但电阻标定有效位、增益、偏移和电缆补偿仍需通过真实标定写入 Flash。分压上下臂方向也必须根据实际原理图确认；如果接法与上图相反，需要修改 ALOG 中的集中方向配置，不能只靠更改 62 Ω 数值解决。

### 1.3 电容和开路检测

当 ADC 结果没有形成 5～50 Ω 电阻候选时：

1. 切换到电容继电器 PG4。
2. TIM5_CH2 在 PA1 捕获 NE555 输出的上升沿。
3. TIM5 计数频率为 2 MHz，即每个 Tick 为 0.5 μs。
4. 第一条边沿只作为起点，随后采集 16 个相邻上升沿周期。
5. 500 ms 内未采满、发生捕获覆盖或定时器错误时，本次检测失败。
6. 对 16 个周期计算最小值、最大值和算术平均值。
7. 使用当前周期与长度检测时保存的开路周期之差换算电容：

```text
delta_tick = 负载周期平均值 - 开路周期平均值
电容(pF) = capacitance_gain_pf_per_tick × delta_tick
           + capacitance_offset_pf
```

8. 标定结果在 50～400 pF 内判为电容，否则判为开路。

当前周期统计使用普通算术平均；最小值、最大值和跨度只用于调试日志，尚未用于异常值剔除。

## 2. BSP 公开接口

### bsp_board.h

| 接口 | 作用 |
| --- | --- |
| `bsp_now_ms()` | 返回 `HAL_GetTick()` 的当前毫秒 Tick。 |
| `bsp_timeout(start_ms, timeout_ms)` | 用 `uint32_t` 无符号减法判断超时，支持约 49.7 天 Tick 回绕。 |

该模块为头文件内联函数，不需要初始化，也不维护额外计数器。

### bsp_path.c/h

| 接口 | 作用 |
| --- | --- |
| `bsp_path_init()` | 强制 PG2、PG3、PG4 全部输出低电平，关闭三路继电器。 |
| `bsp_path_select(path)` | 同步选择 NONE、长度、电阻或电容通路；先全部关闭，等待 10 ms，再只接通一路并等待 10 ms。 |
| `bsp_path_off()` | 立即关闭三个继电器。 |

继电器控制为高电平有效。`bsp_path_select()` 是约 20 ms 的短时有界阻塞函数，只应在主循环调用。

### bsp_load.c/h

| 接口 | 作用 |
| --- | --- |
| `bsp_load_init()` | 停止可能残留的 ADC/TIM5 操作并清空周期捕获现场。 |
| `bsp_load_read_adc(samples, count)` | 检查电阻通路互斥状态，丢弃 4 次 ADC 转换后读取 1～64 个原始样本。 |
| `bsp_load_capture_periods(periods, count, timeout_ms)` | 检查电容通路互斥状态，在指定超时内捕获 1～16 个 NE555 周期。 |
| `bsp_load_get_capture_tick_hz()` | 根据 APB1 时钟和 TIM5 预分频器返回实际捕获 Tick 频率；当前为 2 MHz。 |
| `bsp_load_on_tim_capture_irq(htim)` | TIM5 中断入口，计算相邻 CCR2 捕获值之差并保存周期；ISR 内不做日志、浮点或阻塞操作。 |

`bsp_load_result_t` 可区分参数错误、通路未就绪、ADC错误、定时器错误、超时和捕获覆盖。

### bsp_length.c/h

| 接口 | 作用 |
| --- | --- |
| `bsp_length_init()` | 复位 GP22、发送上电复位命令、写入 7 个配置寄存器、读回 Reg1 高字节并检查 INTN。 |
| `bsp_length_measure_raw(raw, status, timeout_ms)` | 检查长度通路，触发一次 TIM1 单脉冲，有限等待 GP22 INTN，检查 STAT 后返回 RES0 原始值。 |
| `bsp_length_on_gp22_irq(gpio_pin)` | GP22 INTN 的 EXTI 转发入口，只设置中断标志。 |

`bsp_length_result_t` 可区分参数错误、未就绪、SPI错误、脉冲错误、超时、无 STOP1 命中和状态异常。

### bsp_flash.c/h

| 接口 | 作用 |
| --- | --- |
| `bsp_flash_load(data)` | 从内部 Flash Sector 11 读取标定记录，校验记录头和 CRC32；失败时不修改调用者数据。 |
| `bsp_flash_save(data)` | 擦除 Sector 11，按 32 位写入一份标定记录，再读回并校验。 |
| `bsp_flash_erase()` | 擦除整个 Sector 11，并检查其恢复为擦除状态。 |

Sector 11 地址为 `0x080E0000～0x080FFFFF`，共 128 KB。保存和擦除会清除该扇区原有内容，不能再把其他数据放入同一扇区。

## 3. 接线要求

所有外部模块必须与 STM32 共地，并确保进入 MCU 的信号不超过 3.3 V。

| MCU引脚 | 功能 | 接线要求 |
| --- | --- | --- |
| PG2 | 长度通路继电器使能 | 接继电器驱动输入，高电平吸合。 |
| PG3 | 电阻通路继电器使能 | 接继电器驱动输入，高电平吸合。 |
| PG4 | 电容通路继电器使能 | 接继电器驱动输入，高电平吸合。 |
| PC0 | ADC1_IN10 | 接 62 Ω参考电阻与被测负载的分压中点，电压必须在 0～3.3 V。 |
| PA1 | TIM5_CH2 | 接 NE555 的 3.3 V 兼容输出，程序捕获上升沿。 |
| PA4 | GP22 SSN | 接 GP22 低有效片选。 |
| PA5 | SPI1 SCK | 接 GP22 SCK。 |
| PA6 | SPI1 MISO | 接 GP22 SO/MISO。 |
| PA7 | SPI1 MOSI | 接 GP22 SI/MOSI。 |
| PE0 | GP22 INTN | 接 GP22 低有效中断输出；程序使用下降沿 EXTI 和内部上拉。 |
| PE1 | GP22 RSTN | 接 GP22 低有效复位。 |
| PE9 | TIM1_CH1 | 接外部高速脉冲驱动/整形电路的逻辑触发输入。 |
| PA9 / PA10 | USART1 TX/RX | HMI 串口，9600 bit/s。TX/RX需要交叉连接。 |
| PB10 / PB11 | USART3 TX/RX | PC调试串口，115200 bit/s。TX/RX需要交叉连接。 |

PG2、PG3、PG4 只能连接继电器驱动器，不能由 MCU GPIO 直接驱动线圈。驱动级应具备合适的续流保护，且三个继电器的触点必须分别连接长度、电阻、电容前端，在同一个同轴接口处保持互斥。

SPI1 当前为主机、8 bit、MSB first、CPOL=0、第二边沿采样，速率约 328.125 kbit/s。

### 长度前端

- PE9 的 3.3 V 单脉冲只能作为外部高速驱动器的触发信号。
- 外部驱动器负责产生适合同轴电缆的快速边沿和幅值，并完成阻抗、限流及保护。
- GP22 的 START/STOP1 与反射信号调理连接不在 MCU 引脚配置中，必须按实际 TDR 模拟前端原理图连接。
- GP22、脉冲驱动、比较/整形前端和 STM32 必须共地。

### 电阻前端

- 当前软件假定 62 Ω参考电阻接 3.3 V，被测负载经同轴电缆接地。
- 62 Ω在被测端接近短路时电流可接近 `3.3 / 62 ≈ 53 mA`，电阻和供电驱动能力需要留有余量。
- 若实际电路是被测负载接 3.3 V、62 Ω接地，必须先调整集中分压方向配置，否则计算结果会错误。

### 电容前端

- 同轴电缆和终端电容应通过 PG4 控制的继电器接入 NE555 定时网络。
- NE555 输出必须满足 STM32F407 的输入电气规格；若 NE555 使用 5 V 供电，应结合芯片数据手册和实物前端确认 PA1 的输入容限，必要时增加电平转换。
- 16 个完整周期及起始边沿需要在 500 ms 内到达。

## 4. 当前使用注意事项

1. 按当前逻辑，必须先执行长度检测，再执行负载检测。
2. 长度检测结束时会采集“开路周期基准”。为了让电容差分有效，此时同轴终端应处于开路基准状态；之后再接入待测负载进行负载检测。
3. 工厂数据目前只确认 62 Ω参考电阻，未自动启用长度、电阻、电容标定有效位。没有有效 Flash 标定时，程序会输出原始调试数据并提示 `Need Cal`，不会伪造物理量结果。
4. 电阻分压方向、GP22 START/STOP1 模拟前端、NE555 定时元件及继电器实际动作时间仍需结合原理图和实物确认。
5. 任意失败或超时路径都会调用 `bsp_path_off()`，保证三路继电器恢复为关闭状态。
